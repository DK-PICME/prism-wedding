## 📧 각 방식별 핵심 요구사항

### 1️⃣ **이메일/비밀번호** (가장 중요!)

**해야 할 일:**
```
1. npm install firebase
2. Auth Context 만들기 (사용자 상태 관리)
3. SignUpPage 연결 (폼 입력 → 회원가입 API)
4. LoginPage 만들기 (이메일/비번 입력 → 로그인)
5. 토큰 localStorage에 저장
6. 로그인 필요한 페이지 보호
```

**추가 기능:**
- 비밀번호 강도 검사 (8자+, 특수문자)
- 이메일 중복 확인
- 로딩/에러 표시

---

### 2️⃣ **비밀번호 찾기** (간단!)

**해야 할 일:**
```
1. Firebase 콘솔에서 이메일 템플릿 커스터마이징
2. PasswordRecoveryPage에 연결
3. 백엔드 API: POST /api/password-reset
   - 이메일 검증
   - Firebase 재설정 이메일 발송
4. 완료 메시지 표시
```

**이메일 템플릿:**
- Firebase가 기본 제공
- 로고, 메시지만 바꾸면 됨
- 다국어 지원 가능

---

### 3️⃣ **구글 로그인** (중간 난이도)

**필수 설정:**
```
1. Google Cloud Console에서 OAuth 2.0 ID 생성
2. Redirect URI 등록: https://yourdomain.com/auth/callback
3. Firebase와 Google 연동
4. npm install @react-oauth/google
```

**구현:**
```
1. SignUpPage에 GoogleLogin 컴포넌트 추가
2. ID Token 획득 → 서버로 전송
3. 서버에서 Firebase 계정 생성/연결
4. 사용자 정보 저장
```

**장점:**
- Firebase 공식 지원 (쉬움)
- 국제 사용자 지원
- 보안이 좋음

---

### 4️⃣ **네이버 로그인** (가장 복잡!)

**필수 설정:**
```
1. 네이버 개발자 센터에서 애플리케이션 등록
2. OAuth 2.0 인증 신청 (심사 필요)
3. Client ID / Secret 발급
4. Redirect URI 등록
```

**구현 (복잡함!):**
```
1. Naver SDK 로드
2. 로그인 버튼 클릭 → 인증창 오픈
3. Authorization Code 획득
4. 백엔드로 Code 전송
5. 서버에서 Code → Access Token 교환
6. Access Token으로 사용자 정보 조회
7. Firebase 계정 생성/연결
8. JWT 토큰 생성 후 반환
```

**주의사항:**
- 심사 기간 3-7일 필요
- PKCE, 상태값 등 보안 처리 복잡
- 에러 처리 많음

## 🎁 가장 쉬운 것부터 시작하기

### **이메일/비밀번호 선택 이유:**

1. ✅ **준비된 것이 많음**
   - UI 완성
   - Firebase 연결됨
   - API 구조 준비

2. ✅ **구현이 직관적**
   - 입력 → 검증 → 저장
   - 복잡한 OAuth 흐름 없음

3. ✅ **모든 사용자가 사용 가능**
   - 소셜 계정 없어도 가능
   - 회사 이메일 사용 가능

4. ✅ **기초가 되는 기능**
   - 다른 로그인의 기반
   - 계정 복구 등에 필요
