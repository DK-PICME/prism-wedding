### 2️⃣ **네이버 로그인 (OAuth)** ⭐⭐⭐ (어려움)

#### 🔧 구현 난이도
- **프론트엔드**: ⭐⭐⭐⭐ (어려움)
- **백엔드**: ⭐⭐⭐⭐ (어려움)
- **총 난이도**: ⭐⭐⭐⭐⭐ (매우 어려움)
- **예상 소요시간**: 8-12시간

#### ⚠️ 추가 요구사항

**필수:**
- [ ] 네이버 개발자 계정 생성 & 애플리케이션 등록
- [ ] Client ID, Client Secret 발급
- [ ] Redirect URI 설정 (https://example.com/auth/naver)
- [ ] Firebase Custom Claims 설정 (OAuth → Firebase 매핑)
- [ ] 서버사이드 토큰 검증 (Cloud Functions)

**복잡한 부분:**
- [ ] PKCE 흐름 구현 (보안)
- [ ] 상태(state) 값 관리
- [ ] 계정 연결 로직 (첫 로그인 vs 기존 계정)
- [ ] 토큰 만료 및 갱신 처리

**선택:**
- [ ] 네이버 사용자 정보 매핑
- [ ] 프로필 사진 자동 가져오기

#### 📋 체크리스트

```
1. 네이버 개발자 센터:
   - [ ] 개발자 가입
   - [ ] 애플리케이션 등록
   - [ ] OAuth 2.0 인증 신청
   - [ ] Client ID/Secret 발급
   - [ ] Redirect URI 등록

2. Firebase 설정:
   - [ ] OAuth 공급자 활성화
   - [ ] Callback URL 설정
   - [ ] 계정 연결 규칙 정의

3. 프론트엔드:
   - [ ] Naver SDK 로드
   - [ ] 로그인 버튼 클릭 → 인증창 오픈
   - [ ] Authorization Code 획득
   - [ ] 서버로 전송

4. 백엔드 (Cloud Functions):
   - [ ] Authorization Code → Access Token 교환
   - [ ] 네이버 API 호출 (사용자 정보 조회)
   - [ ] Firebase 계정 생성 또는 연결
   - [ ] JWT 토큰 생성 후 반환
```

#### 🔗 네이버 로그인 흐름

```
┌─────────────┐
│   앱 실행    │
```

### 3️⃣ **구글 로그인 (OAuth)** ⭐⭐ (상대적으로 쉬움)

#### 🔧 구현 난이도
- **프론트엔드**: ⭐⭐ (중간)
- **백엔드**: ⭐⭐⭐ (중간)
- **총 난이도**: ⭐⭐⭐ (중간-어려움)
- **예상 소요시간**: 6-8시간

#### ✅ 추가 요구사항

**필수:**
- [ ] Google Cloud Console 프로젝트 생성
- [ ] OAuth 2.0 클라이언트 ID 생성 (웹 애플리케이션)
- [ ] Authorized redirect URIs 설정
- [ ] Firebase와 Google 연동
- [ ] Google Sign-In 라이브러리 설치

**장점:**
- Firebase가 공식 지원 (더 간단함)
- Naver보다 구현이 직관적
- 토큰 검증이 더 안전함

#### 📋 체크리스트

```
1. Google Cloud Console:
   - [ ] 프로젝트 생성
   - [ ] OAuth 2.0 동의 화면 설정
   - [ ] 클라이언트 ID 생성
   - [ ] Authorized redirect URIs 등록
     (예: https://example.com/auth/callback)

2. Firebase 설정:
   - [ ] Google Sign-In 제공자 활성화
   - [ ] 웹 SDK 설정

3. 프론트엔드:
   - [ ] @react-oauth/google 설치
   - [ ] GoogleOAuthProvider 감싸기
   - [ ] GoogleLogin 컴포넌트 사용
   - [ ] ID Token 획득

4. 백엔드:
   - [ ] ID Token 검증
   - [ ] Firebase 사용자 정보 가져오기 또는 생성
   - [ ] JWT 토큰 생성
```

### 4️⃣ **비밀번호 찾기** ⭐⭐ (중간)

#### 🔧 구현 난이도
- **프론트엔드**: ⭐ (쉬움)
- **백엔드**: ⭐⭐⭐ (중간)
- **총 난이도**: ⭐⭐ (중간)
- **예상 소요시간**: 2-4시간

#### 📧 이메일 발송 관련 추가 요구사항

**필수:**

1. **Firebase Authentication 기본 기능 사용**
   ```
   firebase.auth().sendPasswordResetEmail(email)
   ```
   - 자동으로 비밀번호 재설정 이메일 발송
   - 사용자가 이메일 링크 클릭 → 비밀번호 변경 페이지
   - ✅ 가장 간단함!

2. **또는 Custom Email 구성 (Firebase Admin)**
   ```
   Cloud Functions + SendGrid/Gmail API
   ```
   - Firebase Admin SDK로 이메일 생성
   - SendGrid 또는 Gmail API로 발송

**선택적 (이메일 템플릿 커스터마이징):**
- [ ] Firebase Console에서 이메일 템플릿 커스터마이징
- [ ] 로고, 색상, 메시지 변경
- [ ] 다국어 지원

#### 📋 체크리스트

```
# Firebase 기본 방식 (추천)
- [ ] Firebase Console → Authentication → Templates
- [ ] "Password reset" 이메일 템플릿 커스터마이징
  - [ ] 발신자 이메일/이름 설정
  - [ ] 로고 추가
  - [ ] 메시지 수정
  - [ ] 취소 버튼 색상 등 커스터마이징

- [ ] 백엔드 (Cloud Functions):
  - [ ] POST /api/password-reset 엔드포인트
  - [ ] 이메일 주소 검증
  - [ ] Firebase 비밀번호 재설정 이메일 발송
  
- [ ] 프론트엔드:
  - [ ] 이메일 입력 폼
  - [ ] 발송 버튼
  - [ ] 성공 메시지 표시
  - [ ] 에러 처리
```
