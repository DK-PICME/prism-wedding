# 설정 페이지 (SettingsPage) - 구현되지 않은 기능 분석

## 📋 구현되지 않은 기능 목록 (우선순위 기준)

### 🟥 우선순위 1: 매우 높음 (핵심 기능)

#### 1. **아바타 변경 기능** ⭐⭐⭐
- **현재 상태**: UI만 있고 기능 구현 안 됨 (251번 라인의 버튼)
- **난이도**: ⭐⭐ 중간
- **상세설명**:
  - 사용자가 프로필 이미지를 업로드할 수 있게 하기
  - Firebase Storage에 이미지 저장
  - Firestore 사용자 정보에 photoURL 저장
  - 이미지 크롭/리사이징 기능 (선택사항)
  
- **구현 예상 시간**: 1-2시간
- **연관 기능**:
  - 파일 업로드 처리
  - Firebase Storage 인증
  - 이미지 미리보기

#### 2. **이메일 변경 기능** ⭐⭐⭐
- **현재 상태**: UI는 비활성화 (268-274번 라인), "이메일 변경은 지원하지 않습니다" 메시지
- **난이도**: ⭐⭐⭐ 어려움
- **상세설명**:
  - 사용자가 등록된 이메일을 변경할 수 있게 하기
  - Firebase Auth updateEmail() 사용
  - 새 이메일 검증 필요
  - 기존 이메일과 충돌 검사
  
- **구현 예상 시간**: 2-3시간
- **주의사항**:
  - Firebase Auth에서 제공하는 메서드 활용
  - 기존 이메일 검증 프로세스와 통합

#### 3. **사용자정의 파일명 규칙** ⭐⭐⭐
- **현재 상태**: select 옵션에 "사용자정의" 있지만 구현 안 됨 (582번 라인)
- **난이도**: ⭐⭐ 중간
- **상세설명**:
  - "사용자정의" 선택 시 입력 필드 표시
  - 파일명 템플릿 변수 설명 (예: {주문번호}, {날짜}, {파일명})
  - 입력한 규칙 Firestore에 저장
  - 정규표현식이나 검증 필요
  
- **구현 예상 시간**: 1시간
- **연관 기능**:
  - 파일명 프리뷰 생성
  - 변수 치환 로직

### 🟨 우선순위 2: 높음 (중요 기능)

#### 4. **언어 설정** ⭐⭐⭐
- **현재 상태**: UI 준비됨 (622-646번 라인), 테마 선택만 있음
- **난이도**: ⭐⭐⭐ 어려움
- **상세설명**:
  - 한국어/영어/일본어 등 언어 선택
  - i18n 라이브러리 (예: i18next) 도입 필요
  - 언어 설정 localStorage 저장
  - 전체 앱에 적용
  - 사용자 설정 Firestore 저장
  
- **구현 예상 시간**: 2-4시간 (라이브러리 설정 포함)
- **신규 라이브러리**: i18next, react-i18next

#### 5. **다크 테마 완성** ⭐⭐
- **현재 상태**: 테마 선택 UI 있음 (630-643번), 적용 로직 있음 (208-226번)
- **난이도**: ⭐⭐⭐⭐ 매우 어려움
- **상세설명**:
  - 현재는 HTML에 'dark' 클래스 추가만 함
  - 전체 앱의 모든 컴포넌트에 다크 테마 Tailwind 클래스 적용
  - 색상 일관성 검토
  - 접근성(contrast) 검증
  
- **구현 예상 시간**: 3-5시간
- **세부사항**:
  - Tailwind dark: 모드 클래스 검토
  - 모든 페이지/컴포넌트에서 bg-white, text-neutral-900 → dark: 대응

#### 6. **활동 로그/로그인 기록** ⭐⭐
- **현재 상태**: 없음
- **난이도**: ⭐⭐ 중간
- **상세설명**:
  - 최근 로그인 시간/IP/기기 표시
  - 비정상 활동 감지 알림
  - 활동 기록 Firestore에 저장
  
- **구현 예상 시간**: 1-2시간

## 💡 개발 팁

### 아바타 변경 구현 시
- Firebase Storage 경로: `users/{uid}/avatar.png`
- 이미지 크기 제한: 5MB
- 지원 형식: JPG, PNG, WebP

### 언어 설정 구현 시
- `localStorage.setItem('language', 'ko')`에 저장
- 앱 시작 시 localStorage에서 읽기
- 변경 시 전체 UI 리렌더링 필요

### 다크 테마 구현 시
- Tailwind 설정: `darkMode: 'class'`
- HTML 루트에 'dark' 클래스 추가/제거
- 모든 색상에 `dark:` 프리픽스 추가 필요
