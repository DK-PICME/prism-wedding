## 🔍 각 기능 상세분석

### 1️⃣ 아바타 변경 (우선순위: 최고 ⭐⭐⭐)

**현재 상황**
```jsx
// 251번 라인 - UI만 존재
<button className="mt-3 px-4 py-2 border...">
  <i className="fa-solid fa-camera mr-2"></i>아바타 변경
</button>
```

**필요 구현**
- 파일 업로드 인풋 추가
- Firebase Storage에 이미지 저장
- Firestore에 photoURL 저장
- 이미지 리사이징 (선택사항)

**예상 시간**: 1-2시간
**난이도**: 중간 (⭐⭐)

---

### 2️⃣ 이메일 변경 (우선순위: 높음 ⭐⭐⭐)

**현재 상황**
```jsx
// 268-274번 라인 - 완전히 비활성화
<input type="email" value={userEmail} disabled>
  이메일 변경은 지원하지 않습니다
</input>
```

**필요 구현**
- 새 이메일 입력 필드 활성화
- 기존 이메일 검증
- Firebase updateEmail() 호출
- 새 이메일 검증 링크 발송

**예상 시간**: 2-3시간
**난이도**: 어려움 (⭐⭐⭐)

---

### 3️⃣ 사용자정의 파일명 (우선순위: 높음 ⭐⭐)

**현재 상황**
```jsx
// 582번 라인 - 옵션은 있지만 기능 없음
<option value="사용자정의">사용자정의</option>
```

**필요 구현**
- "사용자정의" 선택 시 추가 입력 필드 표시
- 템플릿 변수 설명 (예: {주문번호}, {날짜}, {파일명})
- 입력값 검증
- Firestore에 저장

**예상 시간**: 1시간
**난이도**: 중간 (⭐⭐)

---

### 4️⃣ 언어 설정 (우선순위: 높음 ⭐⭐⭐)

**현재 상황**
```jsx
// 622-646번 라인 - 테마만 있고 언어는 없음
<div>
  <h2>언어 및 테마</h2>
  {/* 테마만 구현됨 */}
</div>
```

**필요 구현**
- i18next 라이브러리 설치 및 설정
- 한국어, 영어, 일본어 등 다국어 지원
- 언어 파일 (JSON) 작성
- 전체 앱에 적용

**예상 시간**: 2-4시간
**난이도**: 어려움 (⭐⭐⭐)

---

### 5️⃣ 다크 테마 완성 (우선순위: 중간 ⭐⭐⭐⭐)

**현재 상황**
```jsx
// 208-226번 라인 - 동작 로직만 있음
if (newTheme === 'dark') {
  html.classList.add('dark');
} // HTML에 클래스만 추가할 뿐 스타일 미적용
```

**필요 구현**
- 모든 페이지 컴포넌트에 `dark:` 클래스 추가
- 색상 일관성 검토
- 접근성(contrast) 검증
- 이미지, 아이콘 다크 모드 대응

**예상 시간**: 3-5시간
**난이도**: 매우 어려움 (⭐⭐⭐⭐)
